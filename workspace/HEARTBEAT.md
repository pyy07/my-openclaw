# HEARTBEAT — 小红书运营方案思考

每轮 heartbeat 时执行以下流程，**不直接操作小红书账号**，只做调研与报告。

## 1. 目标

- 思考**小红书运营方向**与可发内容思路。
- 整理一份简短的「小红书热点 / 运营思路报告」。
- 将报告**发到用户的飞书**（用 message 工具，发给 last 会话对应的飞书用户）。

## 2. 信息来源（多渠道）

运营思路**不要只盯小红书**，从多通道获取灵感：

- **网页搜索**：搜「小红书 热门」「小红书 爆款」「小红书 趋势」以及你关心的垂类（如科技、生活、读书、旅行等）。
- **科技/资讯**：用网页搜索查当日或近期科技、产品、生活方式热点，再转化为「适合在小红书做的选题」。
- **其他平台趋势**：可搜微博热搜、知乎热榜、抖音热点等关键词，提炼成「可迁移到小红书的选题或话术思路」。
- 若已有 **workspace 内笔记、待办、记忆**，可结合用户兴趣做「本周可发内容建议」。

每次不必覆盖所有渠道，选 2～3 类即可，保证报告简短可读。

## 3. 报告结构（示例）

- **一、近期热点/趋势**：2～5 条，每条一两句话 + 可延伸的小红书选题角度。
- **二、运营思路建议**：1～3 条可执行的想法（例如：跟某个热点发测评、做某类合集、蹭某话题做教程）。
- **三、可选**：若当次有特别值得跟的 1 个话题，单独写一句「本期可优先考虑：xxx」。

每条尽量带「为什么适合小红书」或「怎么发」的简短说明。

## 4. 输出与发送

- 将上述内容整理成一条**连贯的正文**（可用 Markdown 列表），通过 **message 工具**发到用户飞书。
- 发飞书时，message 的 to 请用：`user:ou_27eafc99a5b351d7df9ca5709e74ea41`（不要用 heartbeat 作为 to）。
- 不要只回复 `HEARTBEAT_OK`，必须输出并发送至少一份报告摘要。

## 5. 不做的事

- **不**在本流程中直接操作小红书（不发帖、不评论、不点赞）。仅输出「可拿去小红书用的思路与热点报告」。
